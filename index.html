微信小程序选修课 - 课前准备

Posted on 2020-07-31 Edited on 2020-08-05 In App Dev Disqus:

课程简介

在2020 TechX的选修课中，微信小程序入门课程会悉数微信小程序开发的重点知识，教授开发流程，并告诉你如何自己学习并开发出更好的小程序应用🥳。

Prerequisite

  • 希望大家有基本编程基础
  • 有接触过 html/css/js is a plus!
  • web dev 的同学们走过路过不要错过

课件和Demo

这个课程将由demo以主，lecture为辅。 所有的项目文件会在github上面。大家在上课之前下载skeleton，有什么问题可以直接下载完成的project进行调试！
Github链接

软件安装

请大家在课程开始之前安装微信小程序开发工具，根据各自的操作系统和平台安装好！❤️

项目配置

大家在进行微信小程序开发的时候，所有的project都需要一个appID，除非使用测试号。在课程中，使用测试号就足够啦！😈

但是假如大家想分享给亲朋好友炫耀一下，或者比较好奇，我们可以申请一个正式的appID。（必须要成年才行！或者抢来爸妈的手机也行）

课程大纲

暂定的课程大纲，在备课过程中会有改动，仅供参考～

  1. 课程简介，相关IDE安装和介绍，微信小程序架构简介，文件种类介绍，项目配置。
    • 课程简介 (5min)
    • IDE安装和介绍(10min)
    • 微信小程序架构介绍 (5min)
    • 文件种类介绍 (5 min)
    • 项目配置 (5min)
    • TechX 详情程序 Demo，build&run, simulator & phone (30min)

  1. 自带组件介绍，内容排版，stylesheet介绍，data的使用方式，触发事件的绑定。
    • 自带组件介绍 (10min)
    • Stylesheet 介绍(5min)
    • Data 使用方式 (10min)
    • 触发事件 (5min)
    • Demo: Calculator (30 min)

  1. 条件渲染，动态UI，列表渲染，列表元素事件绑定和元素data
    • 条件渲染 (5min)
    • 动态UI (5min)
    • Demo (10 min)
    • 列表渲染 (5 min)
    • 列表元素事件和data (15min)
    • Demo (20 min)
  2. 使用WeUI库，自定义组件，dark mode，更多debug技巧，云开发和云函数
    • WeUI 库简介(10min)
    • 自定义组件 (15min)
    • Dark mode，使用npm模块等 ( 10min)
    • Debug技巧 (5 min)
    • 云开发和云函数 (20min)

Linux command snippets

sudo -E

The -E (preserve environment) option indicates to the security policy that the user wishes to preserve their existing environment variables. The security policy may return an error if the -E option is specified and the user does not have permission to preserve the environment.

Disk space management

df command – Shows the amount of disk space used and available on Linux file systems.

du -a command – Display the amount of disk space used by the specified files and for each subdirectory.

Processes

python main.py >> main.log & - Run main.py

ps aux | grep process_name find the process named process_name

Jupyter

ipython kernelspec list - find ipython kernel list

I’m trying to implement a central control backend for a distributed system called EMQX. To deal with asynchronous task / message queueing, I’m using Python’s threading module to achieve asyc messaging.

Difference between multiprocessing and threading

The short takeaway from my research is that multiprocessing is for CPU-intensive tasks while threading is for IO-intensive tasks. In my case, I need the network communication to be as fast as possible, which means I’m using threading.
