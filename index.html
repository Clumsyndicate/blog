C++ linking

Posted on 2022-11-03 Disqus:

Static linking

  • Larger executable
  • Easier to deploy
  • Imperceptible speed-up

Dynamic linking

  • Places calls at runtime to shared code libraries
  • Smaller executable (reduction in disk space and memory consumed)
  • Improved multitasking for the system (reduced resource consumption)
  • Benefits from library upgrades without re-compilation

Takeaway

Microsoft recommends using dynamic linking whereever possible

Python program as a Service

Posted on 2020-09-23 In Software Disqus:

The stackexchange link.

Summary of my problem: pkill -f cannot kill a running python program reliably, so I have to type it out in command line every time I restart it and check whether multiple processes have been spawned using ps aux | grep *.py.

This is not a satisfactory answer, as I cannot find out the root cause of why pkill -f does not work in a script. I ended up using a systemd Service file to manage my python process. Here’s an example fyi.

Name the file main.service and place it in /lib/systemd/system/

  • Running the service systemctl start main.service
  • Stop the service systemctl stop main.service
  • Restart the service systemctl restart main.service
  • Show status and output systemctl status main.service -l

Now I don’t have to worry about multiple processes running. If the program dies it’ll even restart.
